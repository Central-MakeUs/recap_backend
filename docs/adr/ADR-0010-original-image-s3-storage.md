# ADR-0010: 원본 스크린샷 이미지 S3 저장 재도입 (ADR-0003 대체)

> Architecture Decision Record. 하나의 중요한 의사결정과 그 이유를 기록한다.

| 항목 | 값 |
| --- | --- |
| 상태 | Accepted |
| 날짜 | 2026-07-13 |
| 관련 | [ADR-0003](ADR-0003-no-original-image-storage.md)(Superseded by this) |

## 맥락 (Context)

ADR-0003은 "원본 이미지를 서버가 받지도, 저장하지도 않는다"고 결정했다.
근거는 (1) 서버 분석이 클라이언트 OCR 텍스트만으로 충분했고, (2) 디자인/
레퍼런스처럼 이미지 자체가 정보인 유형이 폐기되어 이미지형 카드 개념이
사라졌기 때문이었다.

이번 "1차 모각작 MVP" API 명세서 설계 과정에서 두 가지가 바뀌었다.

1. 정보카드 상세 화면(07-01)에 `originalImageUrl` 필드와 "원본 이미지
   로딩 실패"(S-07-01) 상태가 명시적으로 포함됨 — 즉 **원본을 다시
   보여주는 화면이 이번 스프린트 범위에 확정됨.**
2. AI 분석 방식이 OCR 텍스트 기반에서 **이미지 자체를 분석하는 방식으로
   전환**됨(별도 결정 사항). 서버가 원본 이미지를 필요로 하게 됨.

즉 ADR-0003이 전제했던 두 조건(서버는 이미지가 필요 없다 / 원본을 다시
볼 화면이 없다)이 모두 무효화되었다.

## 결정 (Decision)

원본 스크린샷 이미지를 **AWS S3에 저장**한다.

- `InfoCard`에 `originalImageKey`(S3 오브젝트 키) 필드를 추가한다.
- 클라이언트가 원본을 조회할 때는 **presigned URL**을 발급해 응답한다
  (버킷을 퍼블릭으로 열지 않는다). presigned URL은 만료 시간이 있으므로,
  `GET /captures/{captureId}`를 재호출하면 새 URL을 받아오는 방식으로
  "다시 시도"를 처리한다(API 명세서 S-07-01에 이미 이렇게 설계됨).
- **보관 기간은 1개월로 임시 설정한다.** 이건 확정된 정책이 아니라
  PM 논의 전까지의 잠정값이다 — 만료 배치 작업을 만들 때 이 값을
  설정으로 분리해두어, 나중에 정책이 바뀌어도 코드 변경 없이 대응할 수
  있게 한다.

## 고려한 대안 (Considered Options)

1. **ADR-0003 유지 (기각)** — 07-01의 원본 이미지 표시 요구사항, 그리고
   AI 분석 방식 전환(이미지 기반 분석) 두 가지 모두와 정면으로 충돌해
   더 이상 유지할 수 없음.
2. **S3 저장 + presigned URL (채택)** — RECAP이 이미 AWS 인프라(EC2,
   RDS)를 쓰고 있어 추가 벤더 도입 없이 확장 가능. presigned URL로
   버킷을 비공개로 유지해 프라이버시 리스크를 최소화.

## 결과 (Consequences)

### 긍정
- 07-01 원본 이미지 표시, AI 이미지 기반 분석 두 요구사항을 모두 충족.
- presigned URL 방식으로 이미지 버킷 자체는 비공개 유지 가능.

### 부정 / 트레이드오프
- ADR-0003이 없앴던 스토리지 비용·프라이버시 관리 부담이 되돌아옴.
- 보관 기간이 미확정 상태(1개월은 임시값)라, 정책이 늦게 확정되면
  이미 쌓인 이미지의 정리 정책을 소급 적용해야 할 수 있음.
